## Project overview

This repository is currently the default Android Studio starter app for a single-module Android application.

- Build system: Gradle Kotlin DSL with a version catalog in `gradle/libs.versions.toml`
- Module layout: one Android application module, `:app`
- UI stack: Jetpack Compose + Material 3
- Package root: `com.example.myapplication`
- Entry point: `app/src/main/java/com/example/myapplication/MainActivity.kt`

## Build and test commands

Use JDK 17 for Gradle commands in this repo. The machine currently also has JDK 24 installed, and Gradle/AGP task creation fails under that JDK.

```bash
export JAVA_HOME="/Library/Java/JavaVirtualMachines/zulu-17.jdk/Contents/Home"
export PATH="$JAVA_HOME/bin:$PATH"
```

From the repository root:

```bash
./gradlew assembleDebug
./gradlew testDebugUnitTest
./gradlew lintDebug
./gradlew connectedDebugAndroidTest
```

Run a single unit test:

```bash
./gradlew testDebugUnitTest --tests "com.example.myapplication.ExampleUnitTest"
```

## Architecture notes

### Build configuration

- Root build logic is minimal: `build.gradle.kts` only applies plugin aliases.
- Repository setup and module inclusion live in `settings.gradle.kts`.
- Dependency and plugin versions are centralized in `gradle/libs.versions.toml`.
- `app/build.gradle.kts` defines the entire Android app configuration: SDK levels, Compose enablement, build types, and test dependencies.

### Application structure

- `MainActivity.kt` hosts the entire app and calls `setContent { ... }` directly; there is no navigation, view model layer, or feature decomposition yet.
- The only UI is a starter `Greeting` composable rendered inside a `Scaffold`.
- Theme setup lives under `app/src/main/java/com/example/myapplication/ui/theme/` and follows the standard Compose template split:
  - `Theme.kt` wires Material 3 color schemes and dynamic color support
  - `Color.kt` defines static palette values
  - `Type.kt` defines typography

### Android app wiring

- `app/src/main/AndroidManifest.xml` only declares the application and launcher activity.
- There are no services, receivers, custom `Application` classes, runtime permissions, deep links, or background components yet.

### Tests

- Local JVM tests live in `app/src/test/`
- Instrumentation tests live in `app/src/androidTest/`
- Current tests are only the generated starter examples, so any real feature work will need its own test structure

## Current state caveats

- This is still scaffold-level code generated by Android Studio; most production architecture decisions have not been made yet.
- The app is configured for very new SDK levels in `app/build.gradle.kts` (`minSdk 35`, `targetSdk 36`, `compileSdk 36`). Verify those values before adding device compatibility requirements.
